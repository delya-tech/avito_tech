
# Тест-кейсы (Test Cases)

### 1. Создание объявления (POST /api/1/item)
- **Цель:** Проверить возможность создания нового товара с полным набором данных.
- **Предусловие:** Генерируется случайный `sellerID`.
- **Ожидаемый результат:** Статус-код 200, в ответе присутствует поле "status".

### 2. Получение всех объявлений продавца (GET /api/1/{sellerID}/item)
- **Цель:** Проверить получение списка товаров конкретного продавца.
- **Входные данные:** `sellerID` = 963791.
- **Ожидаемый результат:** Статус-код 200, тело ответа является списком (JSON Array).

### 3. Получение объявления по ID (GET /api/1/item/{id})
- **Цель:** Проверить получение детальной информации о товаре по его UUID.
- **Входные данные:** Валидный UUID объявления.
- **Ожидаемый результат:** Статус-код 200 (или 404, если товар удален), отсутствие ошибки 400 (Bad Request).

### 4. Получение статистики объявления (GET /api/1/statistic/{id})
- **Цель:** Проверить корректность отдачи счетчиков (лайки, просмотры).
- **Ожидаемый результат:** Статус-код 200, JSON содержит числовые поля статистики.
