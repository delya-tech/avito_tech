# Баг-репорты (Bug Reports)

## Баг #1. Неинформативное сообщение об ошибке при отсутствии поля статистики
- **Описание:** При создании объявления через POST-запрос, если не передать вложенный объект `statistics` с полем `likes`, сервер возвращает 400 Bad Request.
- **Ожидаемый результат:** Либо поле `likes` должно быть необязательным (значение по умолчанию 0), либо документация должна явно указывать на строгую обязательность вложенного объекта.
- **Фактический результат:** Тело ответа: `{'result': {'message': 'поле likes обязательно', ...}}`.

## Баг #2. Несоответствие формата ID в документации и API
- **Описание:** При попытке получения товара по целочисленному ID (например, "1"), сервер возвращает 400 Bad Request с ошибкой валидации UUID.
- **Ожидаемый результат:** Если API поддерживает только UUID, это должно быть строго регламентировано. При передаче неверного формата ожидается 404 или более подробное описание схемы в документации.
- **Фактический результат:** Ошибка `ID айтема не UUID`.
